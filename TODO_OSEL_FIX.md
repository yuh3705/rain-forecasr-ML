# TODO: Fix OSEL Time-Series Stacking

## Current Problem

The current `TimeSeriesStackingRegressor` in `src/aq_course_ml/osel.py` converts `X` to a numeric NumPy array:

```python
X = self._as_array(X)
```

This is not suitable for the current rainfall dataset because `X` includes the categorical column:

```python
city
```

The project uses a preprocessing pipeline (`build_preprocessor`) to handle:

- numeric imputation
- numeric scaling
- categorical imputation
- one-hot encoding for `city`

Therefore, OSEL should keep `X` as a DataFrame and let each base model pipeline handle preprocessing.

## Desired OSEL Design

Use tuned base model pipelines as OSEL base learners:

- Ridge
- Random Forest
- Extra Trees
- LightGBM
- XGBoost

Each base learner should be a full sklearn `Pipeline`:

```python
Pipeline([
    ("preprocess", build_preprocessor()),
    ("model", estimator),
])
```

## Important Fixes

1. Do not convert `X` to NumPy float array inside `TimeSeriesStackingRegressor.fit`.
2. Do not use raw estimators directly as base models.
3. Build or load full pipelines for each base model.
4. Use `.iloc` when indexing DataFrames inside `TimeSeriesSplit`.
5. Set `passthrough=False` by default to avoid passing raw categorical features into the meta learner.
6. Train meta learner only on out-of-fold predictions generated by time-ordered folds.
7. Refit all base pipelines on the full train set after meta learner training.

## Correct Time-Series Stacking Logic

For each `TimeSeriesSplit` fold:

```text
train base models on past fold
predict validation future fold
store out-of-fold predictions
```

Then:

```text
train meta learner on out-of-fold predictions
refit base models on full training set
predict 2026 test set
```

## Why Not Use sklearn StackingRegressor Directly

`StackingRegressor` can accept pipelines, but it internally relies on `cross_val_predict`.

For time series, `TimeSeriesSplit` does not create validation folds that cover every row as a normal partition, so `cross_val_predict` is not appropriate.

That is why the project needs a custom `TimeSeriesStackingRegressor`.
